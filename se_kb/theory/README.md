# 📘 theory/ — 建模基础理论（稳定不变）

`theory/` 目录存放所有与模型语言、软件工程方法论相关的**基础理论**，属于长期稳定、不随具体项目变化的知识。

这些内容定义了“为什么要这样建模”与“模型背后的逻辑基础”。

## 包含内容

- **structured_analysis.json**  
  结构化分析（SA / DFD）的基础理论，包括：系统工程、问题分解、需求分析框架、分析模型组成等。

- **system_modelling.json**  
  各种建模方法论的通用基础（如结构化分析、面向对象、事件驱动等）的原则与概念。

- **semantics.json**  
  建模术语、元素语义、结构含义的解释。例如：process 的语义边界，data store 的逻辑含义等。

## 适用场景
- 给 AI/Agent 提供底层知识支撑  
- 保证不同图类型之间具有一致的理论基础  
- 作为课程、教材、教科书中提炼的可复用内容  

理论层禁止出现任何与具体项目绑定的内容。
